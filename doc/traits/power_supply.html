DrawNotification

DrawNotification represents a consuming device informing a supply device that it intends to draw current.

Callers can use this to help make a power supply more resilient to multiple consumers that might be adjusting

their draw at the same time.

Min Draw Example

Given a notification of {max_draw: 50A, min_draw: 10A} applied to a device with only 30A free.

The device may decide to respond as if the notification was for exactly 30A,

as 30A is within the [10,50] amp range specified in the request.

If no min_draw was specified the device may have responded with an error indicating that

the value of max_draw:50A isn't available.

FieldTypeLabelDescription
id string

Identifies the notification of draw. Required for update or delete operations, should be absent for add operations

max_draw float

The maximum draw that the caller is expecting to consume. In amps. During read, or as part of a write response, this is the maximum draw the supply expects the caller to consume.

ramp_duration google.protobuf.Duration

How long does the caller expect it will be before the draw level has reached its maximum. The device is free to forget draw notifications after this duration has elapsed as it is expected the caller is actually drawing the current so it is not available anymore. If absent or outside the devices supported range during write, the device is free to choose a value. The chosen value will be returned in responses to the write.

min_draw float

The minimum draw that the caller will accept. A zero value indicates that the caller only draws using max_draw and no less. During write, a non-zero value is an indication to the device that it may choose an expected draw value somewhere between the min_draw and max_draw, typically based on free and unallocated current. The max_draw of the response will inform the caller which value has been chosen. See message docs for more details.

PowerCapacity

PowerCapacity describes how much power is available to be supplied by a device.

This can also describe how much isn't available.

Note that load + free <= rating (when rating is present).

When rating is absent it is acceptable to estimate it using load + free for most cases.

The device may only make some of the total power available for consumption.

Notified draw is not accounted for as part of the free calculation.

If it were, during the ramp period the device would count the notified load + the actual load

and potentially double count how much capacity is being used.

FieldTypeLabelDescription
rating float

What is the maximum rating the device can ever supply. Unlikely to change without changes to the electric circuit. In amps. Zero means absent, not a rating of 0 amps.

voltage float

The voltage level the device supplies. In volts. Zero means absent, not 0 volts.

load float optional

How much power is currently drawn from this device. In amps.

capacity float optional

How much of the rated capacity can be supplied from this device. This will update as available capacity changes. In amps. For example a 100A rating with 30A load is likely to respond with 70A capacity.

free float

How much more capacity can be drawn from this device. In amps. Notification of draw should _not_ be included in the calculation for this value. For example a 100A rating with 30A load with a 10A configured safety margin is likely to respond with 60A.

notified float

A total for how much current (in amps) consumers of this supply have notified they will be drawing. See DrawNotification.

PowerSupplyApi

Trait for devices that can supply power, typically electrical.

Method NameRequest TypeResponse TypeDescription
GetPowerCapacity GetPowerCapacityRequest PowerCapacity

Get the current capacity available for supply. The available capacity shouldn't include any draw made on this device directly.
PullPowerCapacity PullPowerCapacityRequest PullPowerCapacityResponse stream

Be notified of changes to the available capacity.
CreateDrawNotification CreateDrawNotificationRequest DrawNotification

CreateDrawNotification indicates that the caller intends to draw the given capacity from this power supply. This device may respond to say that the requested draw is not available at this time.
UpdateDrawNotification UpdateDrawNotificationRequest DrawNotification

Update an existing draw notification. If no such draw notification exists, will return an error.
DeleteDrawNotification DeleteDrawNotificationRequest .google.protobuf.Empty

DeleteDrawNotification allows the caller to remove their notification of draw. Typically used if the situation has changed and the expected draw will no longer happen. If the expected draw level has changed consider using UpdateDrawNotification. If no such draw notification exists, does nothing. There is no need to remove notifications after their ramp_duration has expired.
